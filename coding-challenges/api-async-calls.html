API Asynchronous Calls

  This project began from experimenting with JSON files. My goal was to download some data from an online MMORPG game I love, Guild Wars 2. The game has an API that you can use to get any data about game items, achievements and personal character data. This would be done in the following order:

  • Get the list of all items (get a JSON file of IDs)
  • For each ID, get all the JSON details for the specific item and save its name
  • Create a dictionary with the item name as key and the ID as value
    • This way I could search an item by its name and with its ID I could get the information I needed with an API call.

      The first step I took was to create some basic classes and methods that are responsible for downloading the JSON files. I also created some more methods to manipulate the files after having downloaded them.

      The second step was to save some time. I had to make around 60000 calls to get all the items' names and that was taking ages to complete. For that reason I created a simple console application that would make all the requests asynchronously and save the final dictionary I wanted to create in a JSON format.

      In order to have more fun, I used the Dependency Injection pattern with an HTTPClientFactory. To prevent any timeouts and errors from stopping the app, I also implemented HTTP retry policies in the JSON downloader functions by using Polly.
